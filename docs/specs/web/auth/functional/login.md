# ログイン機能仕様

## 概要

- ニックネームまたはメールアドレスとパスワードでログインする。
- NextAuth.js の `Credentials` プロバイダーと JWT セッションを使用する。
- クライアントからは `loginAction`（next-safe-action）経由で `signIn` を呼び出す。

## リクエスト

| パラメータ名 | 型     | 必須 | 概要                             |
| ------------ | ------ | ---- | -------------------------------- |
| identifier   | string | 必須 | ニックネームまたはメールアドレス |
| password     | string | 必須 | パスワード                       |

クライアント側では `authSchema`（`@/lib/auth-schema`）によりバリデーションする。

## 処理内容

### `authorize`（Credentials プロバイダー）

- `authSchema` でクレデンシャルを検証する。
- `employees` テーブルから `email` または `nickname` が `identifier` と一致するレコードを1件取得する。
- 存在しない場合、エラーを投げる（最終的にログイン失敗として扱われる）。
- `isAccountLocked` が真の場合（`unlockAt` が現在より未来）、`AccountLockedError` を投げる。
- bcrypt でパスワードを照合する。
- 不一致の場合、`recordFailedLogin` で失敗回数を更新し、エラーを投げる。
- 成功時、`resetLoginAttempts` で失敗回数とロックをリセットし、`{ id }` を返す。

### ブルートフォース対策（`@/lib/brute-force`）

- 失敗回数が閾値（5回）に達すると、`unlockAt` を現在から30分後に設定する。
- ロック中はログインを拒否する。

### `loginAction`

- `signIn("credentials", { identifier, password, redirectTo: "/" })` を実行する。
- `AccountLockedError` の場合、サーバーエラーとして「アカウントがロックされています。30分後に再試行してください。」を返す。
- `AuthError` の場合、「メールアドレス/ニックネームまたはパスワードが正しくません」を返す。

### セッション

- JWT 戦略。`session.user.id` に `token.sub` を設定する。
- セッション最大有効期限は 1 日。

## レスポンス

- 成功時：`redirectTo`（`/`) へリダイレクトする。

## エラーハンドリング

| 条件                 | ユーザー向けメッセージ（通知）                                 |
| -------------------- | -------------------------------------------------------------- |
| バリデーションエラー | フォームの各フィールドに `authSchema` のメッセージを表示する   |
| 認証失敗（一般的）   | `メールアドレス/ニックネームまたはパスワードが正しくません`    |
| アカウントロック中   | `アカウントがロックされています。30分後に再試行してください。` |
