## 目的

変更差分のテスト漏れ・観点不足を洗い出す。リポジトリルートで **毎回 `pnpm test:coverage`** を実行し、コンソールと `coverage/` を見る。行より **振る舞い・契約** を優先し、差分と照らす。

## スコア（0–100）

| 範囲   | 説明                                                                           |
| ------ | ------------------------------------------------------------------------------ |
| 90–100 | 🔴 **Critical** — データ損失・セキュリティ・システム障害に直結しうる経路の抜け |
| 75–89  | 🟡 **Important** — ユーザー向けエラーや重要な業務ロジックの抜け                |
| 60–74  | 🟢 **Low** — 混乱や軽微な不具合につながりうる境界・エッジ                      |
| 0–59   | 出力に含めない（補完的・任意の改善）                                           |

**閾値:** **スコア 60 未満**の指摘は出力に含めない。

## 出力形式

### 各指摘

- **スコア**: 数値（60–100）
- **分類**: `Critical` / `Important` / `Low`（🔴 / 🟡 / 🟢）
- **対象**: 実装ファイル `パス:行番号`（テストファイルではない）
- **不足内容**: 何が検証されていないか
- **回帰リスク**: 防ぎうる不具合の例（1 文）
- **追加するテスト案**: 正常 / 異常 / 境界など

分類は **Critical** → **Important** → **Low** の順、同順位なら **スコア** の高い順。指摘ゼロなら「✅ 報告すべき指摘はありません。」のみ。
