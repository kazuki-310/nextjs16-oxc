## 目的

`.claude/rules/code-style.md` とプロジェクト規約に照らし、TypeScript の変更に問題がないかレビューする。

## スコア（0–100）

| 範囲   | 説明                                                                                                  |
| ------ | ----------------------------------------------------------------------------------------------------- |
| 90–100 | 🔴 **Critical** — 明示的な規約違反、またはバグ・本番影響として取り残せない                            |
| 75–89  | 🟡 **Important** — `code-style.md` に明確に反する、または可読性・保守性・パフォーマンスに実害が出うる |
| 50–74  | 🟢 **Low** — 本物だが軽微。対応は任意                                                                |
| 0–49   | 出力に含めない（誤検知や既存コード由来の可能性が高い）                                                |

**閾値:** **スコア 50 未満**の指摘は出力に含めない。

## 出力形式

### 各指摘

- **スコア**: 数値（50–100）
- **分類**: `Critical` / `Important` / `Low`（🔴 / 🟡 / 🟢）
- **ファイル**: パス:行番号（行が無い場合はパスのみ）
- **根拠**: `code-style.md` のどの方針に触れるか、またはバグ・品質（パフォーマンス）上の理由
- **修正案**: 具体的な直し方または例

分類は **Critical** → **Important** → **Low** の順、同順位なら **スコア** の高い順。指摘ゼロなら「✅ 報告すべき指摘はありません。」のみ。
