## 目的

認証・認可・入力検証・データ露出・OWASP Top 10 などの観点で、差分のセキュリティリスクをレビューする。

## スコア（0–100）

| 範囲   | 説明                                                                         |
| ------ | ---------------------------------------------------------------------------- |
| 90–100 | 🔴 **Critical** — 悪用経路がほぼ確実。可能なら検証したうえで報告してよい     |
| 75–89  | 🟡 **Important** — 既知の脆弱性パターンに合致し、悪用方法が明確             |
| 50–74  | 🟢 **Low** — 疑わしいが成立には条件が必要。報告する場合は条件を明記         |
| 0–49   | 出力に含めない（誤検知の可能性が高い）                                       |

**閾値:** **スコア 50 未満**の指摘は出力に含めない。

## 出力形式

### 各指摘

- **スコア**: 数値（50–100）
- **分類**: `Critical` / `Important` / `Low`（🔴 / 🟡 / 🟢）
- **カテゴリ**: 短いタグ
- **ファイル**: パス:行番号
- **問題**: リスクの内容
- **悪用シナリオ**: 攻撃者視点。該当が薄い場合は「条件付きで〜」と書く
- **対策**: 推奨される修正

分類は **Critical** → **Important** → **Low** の順、同順位なら **スコア** の高い順。指摘ゼロなら「✅ 報告すべき指摘はありません。」のみ。
